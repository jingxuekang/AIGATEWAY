## 🔧 修复步骤

### 第一步：进入编辑模式

1. 打开你的 Grafana Dashboard
2. 点击右上角的 **齿轮图标 ⚙️** 或 **"Edit"** 按钮
3. 现在所有面板都可以拖动和编辑了

---

### 第二步：移动 86.5 面板

#### 方法 1: 拖动（推荐）

1. **找到右下角显示 86.5 的面板**
2. **鼠标移到面板顶部**（会出现拖动手柄）
3. **按住鼠标左键不放**
4. **拖动到顶部第一行**，放在最右边（86.7 旁边）
5. **松开鼠标**
6. **调整宽度**：拖动面板右边缘，让它和其他面板一样宽

#### 方法 2: 使用面板设置

1. 点击 86.5 面板的标题
2. 选择 **"Edit"**
3. 在右侧找到 **"Panel options"**
4. 调整 **"Position"** 设置
5. 点击 **"Apply"**

---

### 第三步：修复 0.07% 显示为 7%

#### 找到问题面板

1. 点击显示 **0.07%** 的面板标题
2. 选择 **"Edit"**

#### 检查查询

在查询编辑器中，你会看到类似这样的查询：

```
# 如果是 Prometheus
rate(some_metric[5m])

# 如果是 Elasticsearch
某个聚合查询
```

#### 修复方案 A: 修改查询（推荐）

如果原始数据是 0.07（表示 7%），需要乘以 100：

**Prometheus 示例：**
```
(query_a / query_b) * 100
```

**具体操作：**
1. 在查询框的末尾添加 `* 100`
2. 例如：`rate(requests_total[5m]) * 100`

#### 修复方案 B: 修改单位

1. 在右侧面板找到 **"Standard options"** 部分
2. 找到 **"Unit"** 下拉框
3. 当前可能是 **"Percent (0.0-1.0)"**
4. 改为 **"Percent (0-100)"**
5. 或者选择 **"None"**，然后在 **"Custom unit"** 中输入 `%`

#### 修复方案 C: 使用 Value mappings

1. 在右侧面板找到 **"Value mappings"**
2. 点击 **"Add value mapping"**
3. 选择 **"Range"**
4. From: 0, To: 1
5. Display text: `${__value} * 100`

#### 应用更改

1. 点击右上角 **"Apply"** 按钮
2. 返回 Dashboard

---

### 第四步：修复 0.09% 显示为 9%

对第二个面板（0.09%）重复第三步的操作。

如果你的查询是 Prometheus，直接在查询后面加：

```

```

## 🐛 常见问题

### Q1: 拖动面板时无法放置

**解决：**
- 确保在编辑模式下（点击右上角 Edit）
- 拖动到空白区域或其他面板之间
- 如果还是不行，先删除一个空白面板腾出空间

### Q2: 修改查询后数值还是不对

**检查：**
1. 原始数据是什么？（0.07 还是 7？）
2. 如果原始数据是 0.07，乘以 100
3. 如果原始数据是 7，不要乘以 100，只改单位

### Q3: 找不到 Standard options

**解决：**
- 在面板编辑界面，右侧有多个标签
- 点击 "Panel options" 或 "Standard options" 标签
- 向下滚动找到 "Unit" 设置

### Q4: 保存后刷新页面又变回去了

**原因：**
- 可能有权限问题
- 或者 Dashboard 是只读的

**解决：**
- 确保你有编辑权限
- 尝试 "Save As" 创建新的 Dashboard
- 检查 Grafana 用户权限
